👑 😈 Outil offensif dédié à l'annuaire Active Directory 🩸😈

J'ai commencé ce projet pour me familiariser avec la programmation Golang.
Je code déjà en python, depuis quelques années maintenant.
Étant intéressé par le code et la sécurité informatique, je me suis lancé sur ce projet, qui est un peu ambitieux.

Initialement, je voulais faire un outil ressemblant à Nuclei pour la partie code collaboratif (https://projectdiscovery.io/), mais pour les environnements AD.
J'aime bien ce concept, où tout le monde peut créer un bout de code sur mesure, et l'intégrer dans la codebase de l'outil, afin que d'autre testeurs s'en servent.

Je me sers souvent de Nuclei, Impacket, NetExec, blodHound. Ils sont incontournables, surtout les trois derniers pour ce qui touche aux protocoles utilisés dans windows.
Donc j'ai pensé, "eh pourquoi ne pas recoder ces outils, avec un langage un peu plus rapide que le python? "
(Python -> interprété ; Go -> compilé)

Pour ceux qui on déjà écrit du code en C/C++ ou Golang ou autre lanagages compilés, ils savent que ce n'est pas si facile. IL faut penser à tout, afin d'éviter les bugs fantômes. J'ai essayé de coder en gardant en tête 3 points:

  • 1. L'efficacité,
  • 2. La facilité d'utilisation,
  • 3. Un style agréable à l'oeil dans le terminal (car pourquoi faire un truc moche)

Je me suis aidé avec les documentations techniques des différents modules Go utilisés. Mais beaucoup de l'IA également, pour la partie interconnexion entre les modules, les appels de fonctions, de classes. L'IA m'a pas mal aidé avec les problèmes causés sur Github, lors des différentes mises en lignes du code. La plateforme n'acceptaient que certaines versions de modules.

J'ai grandement sous-estimé la partie maintenance du code dans le temps
Conclusion: J'ai revu mes ambitions à la baisse.
Sur ce projet, en tout cas!

⚒️ Outil Python 🐍🐍

📜 Développement d'un script Python pour automatiser la collecte d'informations système, à ma sauce. Et un peu de couleur. 📜 (réseaux, protocoles etc.) J'y ai inclus une option, pour déterminer les services en cours de fonctionnement. De là à dire que j'ai plagié Nmap, il n'y a qu'un pas, que j'ai franchi allègrement. Une gestion des threads avec le threading. Pour que le programme, scan des ports de manière parallèle. Puis j'ai découvert et testé, le module 'concurrent futures'. Histoire de comparer avec threading. En lançant le script avec l'option 'concurrent', j'ai trouvé le scan plus rapide qu'avec l'option threading. Mais il faut voir ce qu'il donnerait, en testant les 65 536 ports d'une machine. Conclusion: J'ai fait ce projet pour mettre en application, ce que j'ai appris sur différentes plateformes de développement. 👨‍🎓👨‍🎓 (Openclassrooms,Labex, CodeCademy, ProfoundAcademy, WayToLearnX, StackOverFlow (évidemment), Reddit aussi, etc.) Merci aux personnes de ces plateformes et aux développeurs qui partagent leurs connaissances et leur passion.
